## 1. ТРЕБОВАНИЯ К СЕРВЕРУ

### Минимальные требования
- **ОС:** Linux (Ubuntu 20.04+ / Debian 11+ / CentOS 8+)
- **RAM:** 2 ГБ минимум, 4 ГБ рекомендуется
- **CPU:** 1 ядро
- **Диск:** 5 ГБ свободного места
- **Python:** 3.10+
- **Доступ в интернет** (для GigaChat API)

### Дополнительно (для OCR PDF-сканов)
- **Tesseract OCR** — если нужно распознавать текст на сканах PDF
  ```bash
  sudo apt install tesseract-ocr tesseract-ocr-rus tesseract-ocr-eng
  sudo apt install poppler-utils  # для pdf2image
  ```

## 2. УСТАНОВКА

### Шаг 1. Клонируйте репозиторий

```bash
cd /opt
git clone https://github.com/ВАШ_ПОЛЬЗОВАТЕЛЬ/avtoritet-bot.git
cd avtoritet-bot
```

### Шаг 2. Создайте виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate
```

### Шаг 3. Установите зависимости

```bash
pip install --upgrade pip
pip install -r backend/requirements.txt
```

### Шаг 4. Создайте структуру папок

```bash
mkdir -p backend/assistant_giga/data
```

### Шаг 5. Скопируйте базу знаний

Поместите файл `rag_service_aiassistance.txt` в папку:
```
backend/assistant_giga/data/rag_service_aiassistance.txt
```

## 3. НАСТРОЙКА

### Шаг 1. Скопируйте файл конфигурации

```bash
cd backend
cp .env.example .env
```

### Шаг 2. Отредактируйте `.env`

```bash
nano .env
```

Заполните следующие поля:

```env
# === ОБЯЗАТЕЛЬНО ===

# Ключ авторизации GigaChat (Basic token)
# Получить в личном кабинете GigaChat: https://developers.sber.ru/gigachat
GIGACHAT_AUTH_KEY=ваш_basic_token_из_кабинета

# Уникальный идентификатор запроса
# Сгенерировать UUID: https://www.uuidgenerator.net/
GIGACHAT_RQUID=ваш-uuid-идентификатор

# Токен для защиты загрузки документов (придумайте сложный)
ADMIN_UPLOAD_TOKEN=придумайте_сложный_токен

# === НАСТРОЙКИ КОРРЕКТИРОВКИ (по умолчанию) ===

# Язык для OCR
ENABLE_OCR=true
OCR_LANG=rus+eng

# Лимиты загрузки файлов
MAX_UPLOAD_SIZE_MB=25
MAX_UPLOAD_FILES=20

# Настройки поиска
SEARCH_TOP_K=4
HYBRID_ALPHA=0.7

# CORS — укажите домены, с которых разрешены запросы к API
ALLOWED_ORIGINS=https://avtoritet34.ru,https://www.avtoritet34.ru,http://localhost:5500
```

### Шаг 3. Где взять GIGACHAT_AUTH_KEY

1. Зарегистрируйтесь: https://developers.sber.ru/gigachat
2. Создайте проект
3. Получите **Client ID** и **Client Secret**
4. Закодируйте в Base64: `echo -n "CLIENT_ID:CLIENT_SECRET" | base64`
5. Результат вставьте в `GIGACHAT_AUTH_KEY`

### Шаг 4. Где взять GIGACHAT_RQUID

Сгенерируйте UUID v4:
```bash
python3 -c "import uuid; print(uuid.uuid4())"
```
Или онлайн: https://www.uuidgenerator.net/

## 4. ЗАПУСК

### Тестовый запуск (для проверки)

```bash
cd /opt/avtoritet-bot
source venv/bin/activate
cd backend
python app.py
```

Если всё настроено верно, вы увидите:
```
✓ GigaChat access token получен
Инициализация векторного хранилища...
Первичная загрузка 1 файлов...
✓ Загружен rag_service_aiassistance.txt: N чанков
Инициализация кеша...
RAG Pipeline инициализирован (GigaChat)
INFO:     Uvicorn running on http://0.0.0.0:8000
```

Остановите: `Ctrl+C`

### Запуск в фоне (prod)

```bash
cd /opt/avtoritet-bot/backend
nohup venv/bin/uvicorn app:app --host 0.0.0.0 --port 8000 --log-level info > /var/log/avtoritet-bot.log 2>&1 &
```

Проверьте, что работает:
```bash
curl http://localhost:8000/health
# Ответ: {"status":"ok"}
```

## 5. НАСТРОЙКА NGINX (REVERSE PROXY)

### Шаг 1. Установите nginx (если нет)

```bash
sudo apt install nginx
```

### Шаг 2. Создайте конфиг

```bash
sudo nano /etc/nginx/sites-available/avtoritet-bot
```

```nginx
upstream avtoritet_backend {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name avtoritet34.ru www.avtoritet34.ru;

    # Статические файлы сайта (HTML, CSS, JS, картинки)
    location / {
        root /opt/avtoritet-bot;
        try_files $uri $uri/ /index.html;
    }

    # API бота
    location /api/ {
        proxy_pass http://avtoritet_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Увеличиваем лимит для загрузки документов
        client_max_body_size 100M;
    }

    # Health check
    location /health {
        proxy_pass http://avtoritet_backend;
    }
}
```

### Шаг 3. Активируйте конфиг

```bash
sudo ln -s /etc/nginx/sites-available/avtoritet-bot /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### Шаг 4. SSL (рекомендуется)

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d avtoritet34.ru -d www.avtoritet34.ru
```

## 6. SYSTEMD СЕРВИС (АВТОЗАПУСК)

### Шаг 1. Создайте unit-файл

```bash
sudo nano /etc/systemd/system/avtoritet-bot.service
```

```ini
[Unit]
Description=Avtoritet AI Assistant Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/avtoritet-bot/backend
Environment="PATH=/opt/avtoritet-bot/venv/bin"
ExecStart=/opt/avtoritet-bot/venv/bin/uvicorn app:app --host 0.0.0.0 --port 8000 --log-level info
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### Шаг 2. Активируйте сервис

```bash
sudo systemctl daemon-reload
sudo systemctl enable avtoritet-bot
sudo systemctl start avtoritet-bot
sudo systemctl status avtoritet-bot
```

## 7. ЗАГРУЗКА ДОКУМЕНТОВ В БАЗУ ЗНАНИЙ

### Через API (рекомендуемый способ)

```bash
curl -X POST http://localhost:8000/api/admin/documents/upload \
  -H "X-Admin-Token: ваш_токен_из_env" \
  -F "files=@/путь/к/файлу.pdf" \
  -F "replace=true"
```

Ответ:
```json
{
  "task_id": "abc123...",
  "status": "queued",
  "files_saved": ["файл.pdf"],
  "message": "Файлы сохранены. Индексация запущена автоматически."
}
```

### Проверка статуса задачи

```bash
curl http://localhost:8000/api/admin/documents/tasks/abc123 \
  -H "X-Admin-Token: ваш_токен_из_env"
```

### Проверка здоровья

```bash
curl http://localhost:8000/health
# Ответ: {"status":"ok"}
```

### Тестовый запрос к боту

```bash
curl -X POST http://localhost:8000/api/chat/ask \
  -H "Content-Type: application/json" \
  -d '{"query": "Сколько стоит замена масла?"}'
```

## 9. УСТРАНЕНИЕ ПРОБЛЕМ

### Ошибка: `GIGACHAT_AUTH_KEY не установлен`

- Проверьте файл `.env` — должен быть в папке `backend/`
- Проверьте, что токен корректный (Base64 от `CLIENT_ID:CLIENT_SECRET`)

### Ошибка: `401 Неверный ADMIN token`

- Убедитесь, что передаёте правильный токен в заголовке `X-Admin-Token`
- Токен должен совпадать с `ADMIN_UPLOAD_TOKEN` в `.env`

### Ошибка: `Файл слишком большой`

- Увеличьте `MAX_UPLOAD_SIZE_MB` в `.env`
- Проверьте, что nginx не ограничивает размер: `client_max_body_size 100M;`

### Бот отвечает «Нет точной информации»

- Проверьте, что база знаний загружена: `curl http://localhost:8000/health`
- Посмотрите логи: `journalctl -u avtoritet-bot -f`
- Убедитесь, что файл базы знаний лежит в `backend/assistant_giga/data/`

### Ошибка SSL-сертификата при запросах к GigaChat

- Это нормально — GigaChat использует самоподписанный сертификат
- В коде уже стоит `verify=False` для работы с ним

### Порт 8000 уже занят

```bash
sudo lsof -i :8000
sudo kill -9 PID_занятого_процесса
```

Или измените порт в systemd-сервисе и nginx-конфиге.
